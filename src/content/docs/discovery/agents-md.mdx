## Summary

Context files such as `AGENTS.md` are part of your discovery surface: a coding agent that opens your repository discovers how to build, test, and operate the project by reading them, and in a public repository they are exposed to anyone — human or agent — browsing the source. Getting them right is a discovery concern. The full authoring guidance is not discovery-specific and lives in the [Context Files](/docs/context-files) section; this page explains why the surface matters and points you there.

## Why It Belongs to Discovery

Most discovery signals — `llms.txt`, structured data, `robots.txt`, well-known endpoints — help an agent find and understand your product from the outside. Context files do the same job from the inside: they are the first thing a coding agent reads when it lands in your repository, and they determine whether it can act correctly without guessing.

For public repositories, these files are also crawlable web content. They advertise, to any agent that fetches the repo, exactly how the project expects to be built and what boundaries apply. Treat them as a deliberate part of what you expose, not as private scratch notes.

## What Belongs Where

- `AGENTS.md` — the tool-neutral baseline every coding agent reads. Commands, tech stack, testing rules, and permission boundaries go here.
- `CLAUDE.md` — a thin Claude Code overlay for model selection, skills, and memory hints. References `AGENTS.md`; never repeats it.
- `.cursor/rules/*.mdc` and `.github/copilot-instructions.md` — tool-specific overrides layered on the same baseline.

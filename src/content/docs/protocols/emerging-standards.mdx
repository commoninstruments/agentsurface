## Summary

This page is a watchlist for draft and early-stage standards in the agent ecosystem. Everything here is unstable: pre-consensus specs, origin trials, single-vendor proposals, or specs defined by adoption rather than a pinned version. Track them; do not build production dependencies on them without a concrete reason.

Standards that have stabilized enough to earn dedicated coverage no longer live here — they moved to their canonical homes and are listed in "Graduated to stable coverage" below.

- Watch, do not depend: WebMCP, Web Bot Auth, A2UI, AG-UI, AIPREF, NLWeb, OASF, ANP
- Payment-network protocols are watchlist-only; substantive agentic commerce protocols get their own page
- Everything on this page carries a status and a date so you can judge staleness

## Draft & Emerging Watchlist

### WebMCP — MCP for the Browser

A W3C proposal co-authored by Google and Microsoft (February 2026) that exposes page-scoped tools to agents through a `navigator.modelContext` browser API, letting a site publish MCP-style tools that run in the page context rather than on a remote server. It shipped a Chrome 149 origin trial, so you can experiment behind a flag, but the API surface is unstable and no cross-browser commitment exists yet. Watch for whether Safari and Firefox engage.

### Web Bot Auth — Signed Bot Identity

An IETF effort: the WebBotAuth working group is chartered, building on RFC 9421 HTTP Message Signatures so that an accountable agent can cryptographically sign its requests and a site can distinguish it from anonymous automation. Cloudflare has run it in production since March 2026, which gives it real deployment signal ahead of a finished RFC. Relevant to anyone deciding how to let "good bots" through bot-management controls.

### A2UI v0.9 — Generative UI Over Agent Transports

A Google generative-UI specification (v0.9, July 2026) for agents that emit UI rather than only text. It is designed to ride existing transports — AG-UI, A2A, and MCP — rather than define a new one. Extremely new: a sub-1.0 spec with essentially no production track record. Track it as a signal of where generative-UI standardization is heading, not as something to adopt.

### AG-UI — Defined by Adoption, Not a Version

AG-UI is an agent-to-UI event protocol that originated at CopilotKit and has no canonical pinned spec version — its status is best read through its SDK adoption. Implementations span Microsoft Agent Framework, Google ADK, AWS Strands, Mastra, Pydantic AI, LangGraph, and CrewAI. The multi-vendor SDK support makes it the current de facto layer for streaming agent state to a UI, but the absence of a versioned specification means the contract can shift under you. Adopt via a maintained SDK, not by hand-implementing the wire format.

### AIPREF — Content-Usage Preferences

An IETF effort to standardize a `Content-Usage` HTTP header (and companion signals) letting publishers express how their content may be used for AI. It is pre-consensus: the working group is still shaping the vocabulary, and nothing is settled. Watch it alongside Content Signals as the AI-usage-preference space converges.

### NLWeb — Losing Mindshare

NLWeb is a Microsoft initiative to make web pages and APIs natively accessible to agents by adding `/ask` and `/mcp` endpoints derived from schema.org markup. As of mid-2026 it has zero releases and is losing mindshare to WebMCP, which addresses a similar browser-side goal with backing from both Google and Microsoft. Included here for completeness and honesty; do not plan around it.

### OASF — Open Agent Schema Framework

OASF defines normalized schemas for describing agent and tool capabilities, with the goal of making capability discovery work like a package registry — search by what a tool does, not who publishes it. The schemas are still under active development and have not converged on a stable version; the surrounding tooling (search indexes, registry operators) remains nascent. A spec to contribute to and monitor, not to build production dependencies on.

### ANP — W3C AI Agent Protocol

ANP, from the W3C AI Agent Protocol Community Group, targets the problem MCP and A2A leave open: how agents establish trusted identity and communicate when they share no central authority. It combines Decentralized Identifiers (DIDs), JSON-LD capability metadata, and end-to-end encryption. It remains a draft with published working drafts but no production reference implementation. Worth monitoring for cross-organization agent networks where no single OAuth provider can anchor trust.

## Payment-Network Protocols (Watchlist Only)

Card networks are publishing their own agent-payment schemes. These are watchlist pointers only:

- **Visa Trusted Agent Protocol** — Visa's scheme for identifying and authorizing trusted agents in card-network purchase flows.
- **Mastercard Agent Pay** — Mastercard's agent-initiated payment framework.

The substantive, foundation-backed agentic commerce protocols (UCP, AP2, x402) are covered on their own dedicated page (dedicated page forthcoming); the card-network schemes above are tracked here until they show durable, cross-vendor adoption.
